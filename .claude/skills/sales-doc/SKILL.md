# /sales-doc — 案件情報 → 営業文書

## このSkillがやること
案件情報を受け取り、4種類のうち指定された文書を日本語で生成し、
`docs/sales/YYYY-MM-DD-{クライアント名スラッグ}-{種別}.md` に保存する。

## やらないこと（重要）
- **メール送信・DM送信・投稿**。生成して保存するまで。送るのは必ず人間
- **金額と納期の確定**。ユーザーが明示した数字以外は書かない。空欄は `【要確認：金額】`『【要確認：納期】』のマーカーにする
- クライアントから受け取った個人情報・秘密情報を、この文書ファイル以外の場所（プロンプト例・Works・コミットメッセージ等）へ転記すること

---

## STEP 1: 入力の確認（質問で止まらない）

| 項目 | 未指定時の扱い |
|---|---|
| 文書タイプ | 文脈から推測。初回の問い合わせなら「初回返信」、要件が固まっていれば「提案書」。判断できない時だけ1回質問 |
| クライアント名 | 「ご担当者様」とし、ファイル名は `client` にする |
| 案件内容 | ユーザー発言から要約。不明部分は文書内に `【要確認：〇〇】` を残す |
| 金額 | `【要確認：金額】` |
| 納期 | `【要確認：納期】` |

**【要確認】マーカーが1つでも残る文書は「下書き」であり、そのまま送れないことを完了報告で必ず伝える。**

## STEP 2: 共通ルール（全文書タイプ）
- 敬体（です・ます）。営業テンションは控えめ、具体性で信頼を取る文体
- 提供できるものだけ書く。**NG例:**
  - 「著作権・商標のクリアランスを保証します」（保証できない）
  - 「修正は無制限で承ります」（無制限は書かない。回数を書くか【要確認】）
  - 「〇〇社様の実績があります」（実在クライアント実績の捏造。Worksはサンプル制作である旨を正直に書く）
  - 「芸能人そっくりのAIモデルも作れます」（肖像権NG）
  - その場での値引き確約
- AI生成物の性質（生成のたびに結果が揺れる・文字が苦手等）を隠さない。修正対応の枠組みで説明する
- 実績紹介は「AIクリエイティブのサンプルを制作・公開しています」としてサイトURLを添える形にする

## STEP 3: テンプレートに沿って生成する

### タイプA: 初回返信メール（問い合わせへの返事）
```markdown
件名: お問い合わせありがとうございます（Banana Technology）

{クライアント名} 様

お問い合わせいただきありがとうございます。
AIクリエイティブスタジオ Banana Technology の {自分の名前【要確認：名前】} です。

{相手の要望の要約を2〜3行。ここは案件ごとに書く}

弊スタジオでは、AI画像・動画生成を用いた
・商品CM映像の企画・制作（15秒〜30秒）
・AIモデルを起用したビジュアル制作
を承っております。制作サンプルはこちらでご覧いただけます:
{サイトURL【要確認：URL】}

つきましては、以下をお聞かせいただけますと、お見積もりと制作案をご用意できます。
1. 商材と、映像の用途（SNS広告 / Web掲載 / その他）
2. ご希望の尺と本数
3. ご希望の納期
4. ご予算感（差し支えなければ）

ご返信お待ちしております。
```

### タイプB: 提案書
```markdown
# ご提案書｜{案件名}
{クライアント名} 様 ／ Banana Technology ／ YYYY-MM-DD

## 1. ご要望の整理
（箇条書き3〜5点。ヒアリング内容を言い換えて確認する）

## 2. 制作するもの
（尺・本数・構成。/cm-plan があればそのカット割り表をここに転載する）

## 3. 制作の流れと役割分担
1. 企画・カット割りのご確認（お客様確認①）
2. AIによる画像・映像生成
3. 編集・仕上げ → 初稿提出（お客様確認②）
4. 修正対応（{回数【要確認：修正回数】}回まで）→ 納品

## 4. お見積もり
| 項目 | 金額 |
|---|---|
| {項目} | 【要確認：金額】 |

## 5. スケジュール
発注確定から {【要確認：納期】} で納品

## 6. 注意事項（必ずこのまま入れる）
- AI生成の特性上、実在の人物・商標を含む表現には対応できません
- 生成映像内には文字を入れず、ロゴ・コピーは編集工程で合成します
- 納品形式: 1080p / MP4 / H.264（他形式はご相談ください）
```

### タイプC: 見積もり
提案書の「4. お見積もり」を独立させた表＋有効期限（発行から30日）＋支払条件【要確認：支払条件】。
**料金マスタ（下記）に値がある項目だけ数字を書き、無い項目はすべて【要確認：金額】。**

#### 料金マスタ（ユーザーが自分で記入して育てる。空欄のうちは全て【要確認】）
| 品目 | 金額 |
|---|---|
| AI CM 30秒 1本 | （未設定） |
| AI CM 15秒 1本 | （未設定） |
| AIモデル画像 1カット | （未設定） |
| 修正 1回追加 | （未設定） |

### タイプD: 納品メール
```markdown
件名: 【納品】{案件名} 完成データのお送り

{クライアント名} 様

お世話になっております。Banana Technology です。
{案件名} が完成いたしましたので、お送りします。

- 納品物: {ファイル名 / YouTube限定公開URL【要確認：URL】}
- 形式: 1080p / MP4 / H.264
- 修正対応: 残り {回数【要確認：残回数】} 回

ご確認のうえ、修正のご希望があれば {期日【要確認：確認期限】} までにお知らせください。
このたびはご依頼いただきありがとうございました。
```

## STEP 4: 保存と報告
1. `docs/sales/YYYY-MM-DD-{クライアント名スラッグ}-{タイプ}.md` に保存（既存ファイルの上書き禁止、同名は `-2`）
2. チャットには本文全文と、**残っている【要確認】の一覧**を表示
3. 最後に必ずこう書く: 「【要確認】を埋めてからご自身で送信してください。こちらからの送信は行いません。」

## チェックリスト（保存前に自己確認）
- [ ] 金額・納期でユーザーが言っていない数字を書いていない
- [ ] 保証・無制限・実績捏造の表現がない
- [ ] 【要確認】の一覧を報告に含めた
- [ ] クライアントの情報をこのファイル以外に書いていない
- [ ] 送信は人間が行う旨を伝えた
